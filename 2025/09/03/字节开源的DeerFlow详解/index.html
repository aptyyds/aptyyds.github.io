字节开源的DeerFlow详解

DeerFlow 是一个基于多智能体系统的深度研究框架 ，主要在于多智能体之间的交互以及协调 。

用mermaid梳理下框架：

技术架构特点

1. 状态管理

  • 基于 LangGraph 的状态机架构
  • 消息传递系统
  • 检查点保存和恢复
  • 支持 MongoDB 和 PostgreSQL 持久化

2. Web 服务架构

  • 后端: FastAPI + Python
  • 前端: Next.js + React
  • 实时通信: Server-Sent Events (SSE)
  • 容器化: Docker + Docker Compose

3. 扩展性设计

  • 模块化组件架构
  • 插件式工具系统
  • 配置驱动的行为定制
  • MCP 协议支持

4. 多模态功能

  • 播客生成: AI 脚本编写 + 音频合成
  • PPT 生成: 自动化演示文稿创建
  • TTS 集成: 文本到语音转换
  • RAG 集成: 私有知识库支持

工作流程

  1. 用户输入 → 协调器接收并分类
  2. 任务识别 → 判断是否需要深度研究
  3. 背景调查 → (可选) 初步信息收集
  4. 计划制定 → 规划器创建结构化研究计划
  5. 人工反馈 → (可选) 人工审核和修改计划
  6. 执行研究 → 研究团队并行执行任务步骤
  7. 报告生成 → 聚合结果并生成最终报告

一个多智能体协调系统 ，首先关注多智能体之间的通信，还有工作流是如何实现的 ，外部的工具调用 ，MCP是否使用 ，如何集成的？

多智能体之间的通信：

首先 ，这个项目用的是LangGraph作为核心通信基础设施 ，基于 LangGraph 的状态机模式，通过共享状态实现智能体间通信

MessagesState作为全局共享状态，通过这个状态进行通信 ，所有智能体可以读取和更新这个共享状态， 实现信息传递。
